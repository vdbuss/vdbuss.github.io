Hoofdstuk 0

Hoofdstuk 1

  • Wat bedoelt men juist met volgende termen?

    Server, workstation, PC:
    Een server is een krachtige computer die voor meerdere gebruikers tegelijk kan werken, of die meerdere taken tegelijk kan uitvoeren. Een PC is een computer voor persoonlijk gebruik. Een workstation is een zeer krachtige PC, die voor bepaalde doeleinden ook als server kan diene; dus een computer die qua kracht in het midden ligt tussen een PC en een server.
    Desktop replacement:
    Normaal werkt men op het bureel op een desktop computer (het typische formaat PC, dus een bak, een scherm, en een toetsenbord), en werkt men op een draagbare computer (meestal laptop of portable genoemd) als men op verplaatsing moet. Meer en meer mensen echter gebruiken 1 enkele laptop als computer, en gebruiken die ook op hun bureau. We spreken dan over "desktop replacement".
    Docking station, port replicator:
    Je mag de term "port replicator" vergeten; het is een synoniem voor de veel meer gebruikte term "docking station".
    Embedded computer:
    Een computer (typisch special-purpose) die ingebouwd zit in een ander toestel. Bijvoorbeeld, moderne wagens zitten propvol embedded computers, bijvoorbeeld voor het ABS. Ook een mobiele telefoon, of een digitale camera, bevat een embedded computer. De meeste moderne apparaten bevatten embedded computers.
    Network computer:
    Een computer speciaal bedoeld om vanop afstand, via het netwerk, te werken op een server (host system, zie Hoofdstuk 8). Een network computer is dus een special-purpose computer, en doet typisch niet veel meer dan op het scherm te tonen wat de host computer doorstuurt, en wat jij op het toetsenbord of met je muis intikt op zijn beurt door te sturen naar de host computer. Network computers zijn nog vrij zeldzaam, zeker in Europa.
    Set-top box:
    Een soort van network computer (zie hierboven) die je aansluit op je TV waardoor je via de server van de kabelmaatschappij op het Internet kan, of op andere netwerken (zoals bijvoorbeeld het netwerk van de kabelmaatschappij zelf waarmee ze video on demand aanbieden).
    Information appliance:
    Verzamelnaam voor network computers specifiek bedoeld om een consument toegang tot het Internet (of andere netwerken) te verlenen zonder een echte PC. Set-top boxes (zie hierboven) zijn dus een bepaald soort information appliances, die via de TV werken. Een mobiele telefoon waarmee je op het Internet kan is een ander voorbeeld van een information appliance.
    High-end computer:
    Een zeer krachtige computer.
    Stripped-down computer, full-blown PC:
    De Engelse term "full-blown" betekent "volledig", dus een full-blown PC is gewoon een PC met alles erop en eraan. De Engelse term "stripped-down" is zowat het omgekeerde van "full-blown": een stripped-down computer is een computer met erg beperkte functionaliteit, zoals een network computer (zie hierboven).
  • Wat bedoelt men met "artificial intelligence"?

    Dat is een verzamelnaam voor alle software die de computer taken laat uitvoeren waar we van uitgaan dat ze een zekere intelligentie vereisen: het begrijpen van taal bijvoorbeeld, of het stellen van een medische diagnose vanuit gegeven symptomen is een ander voorbeeld. Artificial intelligence is een groot wetenschappelijk gebied dat verbanden heeft met bijna alle andere wetenschappen.

  • Is Powerpoint een voorbeeld van "desktop publishing"?

    Neen. Desktop publishing is software gebruikt in de uitgeverijsector (zoals kranten of magazines). Hier wordt dieper op ingegaan in hoofdstuk 5 van het boek.

  • Wat moeten we juist kennen over sociale en ethische kwesties?

    Die discussie behoort wel degelijk tot de leerstof, maar je hoeft hiervan niet elk detail van buiten te leren.

Hoofdstuk 2

  • Wat bedoelt met met volgende termen?

    Internal clock:
    De processor voert instructies uit met de regelmaat van een klok. Moderne processoren voeren meer dan een miljard elementaire instructies uit per seconde. De architectuur van de processor bevat dan ook daadwerkelijk een electronische klok die dit synchroniseert.
    Volatile memory, ROM, firmware:
    Volatile memory is geheugen waarvan je de inhoud kwijt bent als je de stroom uitschakelt. Bijvoorbeeld, het normale inwendige geheugen van een PC (het RAM genoemd) is volatile (Nederlands: vluchtig). Maar in een computer zitten ook een aantal geheugenchips die nonvolatile (niet vluchtig) zijn. Zo bijvoorbeeld het ROM, of recenter het flash memory dat het voordeel heeft op ROM dat de inhoud nog gewijzigd kan worden. Nonvolatile geheugen bevat dikwijls "firmware": dit is software die vast in een nonvolatile geheugenchip van de computer is opgeslagen. Een voorbeeld van firmware in een gewone PC is het opstartprogramma voor wanneer de computer wordt ingeschakeld. In een GSM (die een embedded computer bevat) is het volledige computerprogramma dat de werking van het apparaat bestuurt, helemaal firmware. Nonvolatile geheugen kan ook nog andere dingen bevatten dan firmware, bijvoorbeeld het adresboek op je GSM.
  • Is een host computer hetzelfde als een server?

    Een host computer is een speciaal soort server waarop men vanop afstand kan werken. "Server" is een veel algemenere term die slaat op elk soort computer die diensten levert aan andere computers.

  • Zijn de "how it works" leeskaders examenstof?

    Zoals alle soorten leeskaders (overzicht op paginas xx een xxi van het boek) zijn deze geen examenstof, wel mogelijk nuttig leesmateriaal. Het is logisch dat in dit technische hoofdstuk, over computer hardware, nogal veel "how it works" kaders voorkomen.

Hoofdstuk 4

  • Wat is een compiler? En is Java niet gewoon een programmeertaal zoals alle andere? En wat zijn applets?

    De processor van de computer kan enkel programma's uitvoeren in de machinetaal van de processor. Deze machinetaal bestaat uit een aantal instructies, die elk een nummer hebben, en een programma in machinetaal is dus een reeks zulke instructies; een reeks getalletjes dus; uiteindelijk, een rij van bits op machineniveau. Als mensen een programma schrijven doen ze dat echter niet in deze machinetaal, omdat dit veel te onhandig is. Wij schrijven computerprogramma's in hogere programmeertalen, zoals Visual Basic, maar er zijn er nog tal van andere, zoals C++, Fortran, Cobol, en nog veel meer. Als we een programma, geschreven bijvoorbeeld in Visual Basic, echter willen laten uitvoeren door de computer, dient de programmatekst nog worden omgezet naar machinetaal. Dit gebeurt door een speciaal systeemprogramma dat "compiler" genoemd wordt. In de Visual Basic programmeeromgeving, waarin we onze programmeeroefeningen maken, is de compiler een belangrijke component. Telkens we ons programma laten runnen wordt de compiler achter de schermen opgeroepen.

    Een buitenbeentje is Java, een hogere programmeertaal die door de Java compiler niet helemaal wordt omgezet in machinetaal, maar wel in instructies van een andere, laag-niveau taal ("Java bytecode" genaamd). Een tweede programma ("Java virtual machine" genoemd) kan zulke Java bytecode-instructies dan uitvoeren. Het voordeel is dat je een Java programma kan laten uitvoeren door eender welke computer waarop zo'n Java virtual machine loopt. Als we ons programma willen kunnen draaien op 3 verschillende types van processoren (bijvoorbeeld een gewone PC, een Mac, een mainframe), hoeven we het dus niet 3 keer te compileren naar de 3 verschillende machinetalen. Het volstaat dat er Java virtual machines beschikbaar zijn voor deze 3 processoren. Dit wordt de "platformonafhankelijkheid" van Java genoemd. Een Java virtual machine kan ook verschillende stukjes Java-bytecode, verspreid op het Internet, samenvoegen en uitvoeren als 1 globaal programma. Daarom zegt men dat Java geintegreerd is met het Internet.

    De meeste Web browsers bevatten zelf een Java virtual machine, zodat we Java-programma's kunnen laten lopen binnenin een browservenster. We noemen zulke Java-programma's "applets". Zulke applets worden veel gebruikt in educatieve websites. Als voorbeeld deze website met een applet die de beweging simuleert van de aarde rond de zon.

  • Wat is het verschil tussen shareware en een demo versie?

    Shareware is software die je vrij kan uitproberen, maar als je echt van plan bent de software te gaan gebruiken verwacht de auteur van de software wel dat je hem betaalt. Een demo versie is eveneens software die je vrij kan uitproberen, maar die niet alle functionaliteiten biedt van het volledige pakket, dat je moet betalen. Sommige demo versies bieden wel de volledige functionaliteit, maar vallen uit na een aantal dagen, behalve als je betaalt. De grens tussen shareware en demo versies is dus niet echt scherp te trekken.

  • Wat is het verschil tussen multitasking en concurrent processing?

    Je mag deze twee termen als synoniemen beschouwen.

  • Wordt het virtueel geheugen opgeslagen op de harde schijf?

    Inderdaad.

  • Behoren device drivers tot het besturingssysteem?

    Het besturingssysteem komt standaard met een heleboel device drivers voor de basisrandapparatuur zoals scherm, toetsenbord, harde schijf, enzovoort. Voor bijkomende randapparatuur die je je aanschaft, bijvoorbeeld een nieuwe printer, heb je dan bijkomende device drivers nodig die niet altijd reeds aanwezig zijn. Je mag device drivers beschouwen als onderdelen van het besturingssysteem.

  • Zijn command-line en menu-driven interfaces character-based?

    Ja, aangezien ze beiden louter met behulp van tekstboodschappen werken.

  • Is UNIX louter een besturingssysteem dat verschillende gebruikers toelaat op de computer? Kan Windows dat dan ook niet?

    Neen, UNIX is meer dan dat, het is gewoon een ander type van besturingssysteem dan Windows. UNIX laat inderdaad toe dat verschillende gebruikers tegelijk op de computer kunnen werken (dankzij concurrent processing, en met behulp van verschillende schermen en toetsenborden uiteraard). Het is juist dat de recentste versies van Windows dit ook toelaten (wordt daar "remote desktop" genoemd).

  • Linux is "open source" software. Betekent dat gewoon dat het gratis aangeboden wordt? Heeft het voor de rest quasi dezelfde mogelijkheden als UNIX?

    Linux is inderdaad erg verwant met UNIX. Dat het "open source" is betekent niet slechts dat het gratis is. Het betekent dat het volledige programma (een besturingssysteem is tenslotte ook een programma, zij het een heel complex) vrij in te kijken is, het is dus geen bedrijfsgeheim hoe Linux in elkaar zit. Iedere capabele computerprogrammeur met een beetje vrije tijd kan aan Linux meewerken. Linux wordt door een hele gemeenschap van programmeurs onderhouden en aangepast.

  • Wat is .NET?

    .NET is het antwoord van Microsoft op Java. Het is een alternatief voor de Java virtual machine, speciaal ontworpen voor het Windows besturingssysteem. Ruwweg dezelfde eigenschappen van de Java virtual machine (hierboven beschreven bij de vraag over compilers en Java) gelden ook voor .NET, zo ook de integratie met het Internet.

  • Wat is een utility program?

    Een programma dat een nuttige bijkomende systeemtaak uitvoert die niet reeds door het besturingssysteem zelf wordt vervuld. Een typisch voorbeeld van een utility program is een virus checker. Vele utility programs komen gebundeld samen met het besturingssyteem, en je kan ze dan eigenlijk beschouwen als deel uitmakend van dat besturingssysteem. De grens tussen de utility programs en het besturingssysteem zelf is dus niet echt scherp te trekken.

Hoofdstuk 5

  • Wat is het verschil tussen "editing" en "formatting" van tekst?

    Met "editing" (Nederlands: redigeren) bedoelt men de tekst zelf aanpassen; met "formatting" bedoelt men het uitzicht van de tekst, dus dingen zoals lettertype, uitlijning, regelbreedte, enz.

  • Bij groupware, laat de "Track Changes" functie van Microsoft Word toe dat aanpassingen gemaakt door 1 persoon direct kunnen gezien worden door alle andere personen?

    Neen, bij gebruik van Word moet het Word document moet nog wel steeds rondgestuurd worden, bijvoorbeeld per email. Meer gesofistikeerde groupware systemen laten wel toe dat verschillende personen tegelijk werken aan een document en dat alle veranderingen onmiddellijk worden verspreid.

  • Bij desktop publishing, wat bedoelt men precies met "typesetting"?

    Typesetting is Engels voor "letterzetten", hiermee bedoelt men het bepalen van de juiste lettertypes, en uiteindelijk ook het afdrukken van de tekst in deze lettertypes.

  • Wat bedoelt men met een "stack chart"?

    "To stack" is Engels voor "stapelen", een stack chart is dus een superpositie van een aantal bar charts boven elkaar, bijvoorbeeld voor verschillende jaren. Enkele voorbeelden vind je in op de website van Nevron.

Hoofdstuk 7

  • Wat is de functie van "hot sync" tussen een PDA en een PC?

    Wanneer je dezelfde gegevens bewaard zowel op je PDA als op je PC, bijvoorbeeld een adressenbestand, en je doet een wijziging op je PC, wil je deze wijziging automatisch laten doorvloeien naar je PDA, en omgekeerd, als je een wijziging doet op je PDA, wil je die laten doorvloeien naar je PC. Dit gebeurd m.b.v. de "hot sync" functie. Een illustratie wordt gegeven in het boek, pagina 247.

  • Wat is het verschil tussen batch processing en interactive processing?

    Bij batch processing gebeurt de gegevensverwerking niet continu, maar in ladingen. Bijvoorbeeld, een bank die alle overschrijvingen tussen rekeningen die overdag gebeurd zijn, 's nachts pas definitief verwerkt in de bestanden. Dit staat tegenover interactive processing (ook wel "on line" of "real time" processing genoemd) waar elke bewerking op de gegevens onmiddellijk wordt uitgevoerd.

  • Wat bedoelt men met "centralized database" op pagina 254?

    Men bedoelt hier dat de gegevens in een centrale database, op één enkele server (typisch een mainframe), zijn opgeslagen, en dat enkel de informatici die de server beheren toegang hebben tot de gegevens. Dit was zo dertig jaar geleden! Tegenwoordig is de database dikwijls nog steeds centraal, maar kunnen de gebruikers via toepassingssoftware op hun PC rechtstreeks met gegevens uit die database werken. Dit wordt verder besproken in het boek op dezelfde pagina.

Hoofdstuk 9

  • Wat bedoelt men met volgende termen?

    Dynamic HTML:
    Dit is enkele een verzamelnaam voor technologieën die gebruikt worden om Webpagina's meer dynamiek te geven dan enkel lezen wat erop staan en klikken op links. Onder deze verzamelnaam vallen bijvoorbeeld CSS en JavaScripts.

    CSS is een technologie die de layout en de user interface van Webpagina's verbetert. Een voorbeeld van een pagina die geen CSS gebruikt is deze pagina! Meer modern ogende websites, bijvoorbeeld die van Canvas, gebruiken volop CSS.

    Een JavaScript is een programaatje dat je Web browser uitvoert terwijl je de Web pagina gebruikt, waardoor de Web pagina interactief wordt. Een voorbeeld is deze EURO calculator. (JavaScript heeft tussen haakjes weinig of niets te maken met Java, de programmeertaal.)

    Gateway:
    Om een LAN (zoals op school) te koppelen aan een WAN (zoals het Internet) is gespecialiseerde apparatuur nodig. Zulke apparaten noemen we bridges (bruggen) of gateways (doorgangen). Je mag deze twee termen als synoniemen behandelen.
