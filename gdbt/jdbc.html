JDBC

Voordat je begint, zorg dat je de db2profile hebt uitgevoerd, zoals beschreven hier.

Inleiding

JDBC is een application programming interface (API) om vanuit een Java programma commando's te versturen naar een database en resultaten hiervan op te vragen en te overlopen.

Handleiding

Het hoofdstuk over JDBC in de DB2 handleiding vind je op de onderstaande links. Je kan kiezen uit twee types van drivers om te gebruiken: type 2 en type 4. De handleiding zegt dat type 2 beter zou zijn als jouw Java programma runt op de server zelf en type 4 als jouw Java programma over het internet heen contact opneemt met de server. Belangrijke aandachtspunten in de handleiding zijn:

  • Het gebruik van de DriverManager om een Connection object te bekomen.
  • Het gebruik van dit Connection object om SQL statements uit te voeren en mogelijke ResultSet-objecten als output te ontvangen (en te overlopen).

Als je gewoon op Spooky de oefeningen maakt, dan kan je kiezen uit de volgende twee url's bij het connecteren:

  • "jdbc:db2://spooky.uhasselt.be:50002/gdbt" (voor type 4)
  • "jdbc:db2:gdbt" (voor type 2; dit geeft geen problemen omdat de naam gdbt op Spooky gekend is)

Het is eigenlijk het handigste om de url jdbc:db2:gdbt te gebruiken omdat je dan bij het connecteren geen paswoord moet ingeven. Dit werkt alvast zeker als je het JDBC programma runt op de Spooky server zelf. Veronderstel een eenvoudige tabel person. Deze JDBC voorbeeld-code haalt resultaten op uit deze tabel zonder dat er een paswoord moet worden ingegeven.
